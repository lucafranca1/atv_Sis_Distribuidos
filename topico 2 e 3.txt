3. Discutir diferenças entre RPC e REST API.

A principal distinção entre RPC e REST API está no modelo de interação: a RPC é focada em Ações e Funções, enquanto a REST é focada em Recursos e Objetos. No RPC, você chama um procedimento específico no servidor, como se estivesse dando uma ordem direta. Na prática, isso geralmente significa usar o método HTTP POST para todas as ações, já que o nome da função é o que realmente importa. Já na REST, você interage com um recurso em um endereço, utilizando verbos HTTP padronizados para descrever a ação que deseja realizar sobre esse recurso. Essa abordagem torna a REST menos acoplada, pois você interage com os objetos de forma uniforme, diferente da RPC, onde o cliente fica mais "grudado" aos detalhes da implementação da função no servidor.
